# 📱 EduManager 화면 상세 기획 (UI Specs)

## 0. 공통 사항
- **디자인 컨셉**: 직관적이고 가독성 높은 UI. 특히 차량 운행 중 흔들림을 고려하여 터치 영역(Button Touch Area)을 넓게 확보.
- **컬러 시스템 (Shuttle Operation)**:
    - **하차 (Drop-off)**: Blue (Default) - 설정에서 변경 가능 (5~8가지 프리셋)
    - **승차 (Pick-up)**: Black (Default) - 설정에서 변경 가능
    - **결석/미탑승 (Exception)**: Red - 표시 여부 옵션 제공

---

## 1. 홈 화면 (Dashboard)
모든 권한(Role)의 사용자가 앱 실행 시 가장 먼저 접하는 화면.

### [VIEW] Dashboard
- **Header**: 학원 로고/이름, 프로필 접근 버튼.
- **Status Cards (Grid Layout)**:
    1. **전체 원생 수**: 현재 등록된 총 원생 수.
    2. **오늘 출석 예정**: 금일 스케줄이 있는 학생 수.
    3. **오늘 결석 예정**: 사전에 '결석' 처리된 학생 수.
    4. **금일 공지사항**: 오늘 날짜의 공지사항 개수.
    - *(제안)* **승인 대기 (Admin)**: 가입 승인 대기 인원 숫자 (Badge).

---

## 2. 차량 기사 앱 (Driver Mode)

### [VIEW] Vehicle Selection (Pre-Drive)
- 앱 진입 혹은 '운행 모드' 진입 시 **차량 선택** (여러 대 운용 고려).
- 차량 선택 -> 노선 선택 -> **[운행 시작]**.

### [VIEW] Shuttle Operation (Driving)
화면을 세로로 분할하여 정보를 계층적으로 표시.

| 영역 | 비율 | 내용 및 기능 |
| :--- | :--- | :--- |
| **Top** | 15% | **이전 정류장**<br>- 딤(Dim) 처리 배경.<br>- 승/하차 인원 요약.<br>- 터치 시 상세 보기(확대) 진입. |
| **Middle** | 55% | **현재 정류장 (Main Focus)**<br>- **Info**: 정류장명 텍스트.<br>- **Stats**: 하차 n명 / 승차 n명 (숫자 강조).<br>- **List**: **[하차 명단]** (Blue) → **[승차 명단]** (Black) 순서로 정렬.<br>- **Exception**: 결석 학생은 Red (표시 옵션 켜짐 시).<br>- **Interaction**: 학생 탭 → Action Sheet/Modal. |
| **Bottom-Mid** | 30% | **다음 정류장**<br>- 정류장명 및 예정 인원 미리보기.<br>- 터치 시 상세 보기(확대) 진입. |
| **Bottom** | Fixed | **Control Bar**<br>- **[< 뒤로]**: 실수로 넘겼을 때 이전 정류장 복귀.<br>- **[완료/이동 >]**: 현재 정류장 처리 완료 및 이동. |

#### ⚙️ Interaction Details
- **정류장 이동 (완료/이동 버튼)**:
    - 터치 시 다음 정류장 화면으로 전환.
    - **Validation Popup**: 상태 변경(결석, 차량 제외 등)된 학생이 있는 경우 팝업 발생.
        > "홍길동 학생이 결석 처리되었습니다. 다음 정류장으로 이동하시겠습니까?" (확인/취소)
- **상세 보기 (Expand Mode)**:
    - 이전/다음/현재 정류장 영역 터치 시 해당 영역이 확장됨 (다른 영역 높이 0% 수렴).
    - 편안한 스크롤 및 학생 개별 제어 가능.
- **학생 제어 (Student Action)**:
    - 학생 탭 시 메뉴: **[결석 처리]**, **[차량 제외]**, **[정보]**.
    - **정보 버튼**: 학생 연락처(보호자) 확인 및 **[통화 걸기]** 기능.
    - **상태 토글**: 이미 '결석'인 학생을 누르면 **[결석 취소]** 메뉴 노출.
    - **메모(Memo)**: 상태 변경 시 메모 입력 (MVP 제외).

## 3. 선생님 앱 (Teacher Mode)

### [VIEW] Daily Timetable
- **View Type**: 시간표(Time-table) 중심 리스트.
- **Content**:
    - 시간대별(예: 14:00, 15:30) 수업 목록.
    - 수업 탭 시 해당 시간대 **출석부 명단** 확장.
- **Feature**:
    - 선생님은 차량의 현재 위치나 학생의 탑승 여부를 굳이 볼 필요 없음 (수업 준비 집중).
    - 단순 출석/결석 현황 체크.

---

## 4. 관리자 앱 (Admin Mode)

### [VIEW] Management
- 모바일 앱 내에서 데이터 등록/수정 기능 지원 (MVP).
- **데이터 등록**:
    - **원생 등록**: 이름, **학교(소속)**, 학부모 연락처.
        - *학교 입력은 **객관식 선택(Selection)** UI로 오타 방지.*
    - **차량/노선 관리**: 정류장 추가 및 순서 편집.
    - **승인 관리**: 가입 요청한 교사/기사 계정 승인.

---

## 5. 전체 화면 리스트 (Screen List)

프로젝트 구현에 필요한 전체 화면 목록입니다.

### 5.1 인증 및 초기 진입 (Auth)
- **로그인 화면**: Google / Apple 소셜 로그인 버튼 제공.
- **초대 코드 입력 화면**: 가입 후 소속 학원 확인을 위한 코드 입력.
- **승인 대기 화면**: 관리자의 승인 전까지 노출되는 안내 화면.

### 5.2 공통 (Common)
- **홈(대시보드) 화면**: 학원 상태 요약 및 지표 표시.
- **설정/마이페이지**: 프로필 관리, 로그아웃, 알림 등 앱 환경 설정.

### 5.3 차량 기사 (Driver Mode)
- **차량/노선 선택**: 운행 전 차량 및 코스 선택.
- **실시간 운행 화면**: 4분할 레이아웃 기반 승하차 체크.
- **학생 상세 모달**: 연락처 확인 및 상태 변경(결석/제외) 처리.
- **운행 종료 리포트**: 당일 운행 결과 요약.

### 5.4 선생님 (Teacher Mode)
- **수업 시간표**: 오늘 담당 수업 리스트.
- **출석 상세 뷰**: 개별 수업의 원생 출결 현황.

### 5.5 관리자 (Admin Mode)
- **관리 대시보드**: 관리 메뉴(사용자/원생/노선) 진입점.
- **가입 승인 관리**: 대기 중인 교사/기사 승인 목록.
- **원생 관리**: 원생 목록 조회 및 등록/수정(학교 선택 포함).
- **차량/노선 관리**: 학원 차량 및 운행 코스 설정(고정/변형).
- **공지사항 관리**: 전체 공지 작성 및 관리.
